# CSS-ის საფუძვლები

## ნაწილი 1: CSS-ის გამოყენების 3 გზა

ვებ-გვერდზე სტილების მოსარგებად სამი ძირითადი მეთოდი არსებობს:

1. **გარე (External) CSS (საუკეთესო პრაქტიკა)**

   ეს მეთოდი ინარჩუნებს HTML კოდის სისუფთავეს და საშუალებას გაძლევთ ერთი ფაილით მართოთ მრავალი გვერდის დიზაინი.

> **როგორ:** დააკავშირეთ `.css` ფაილი HTML-ის <head> სექციაში link ტეგით.

```HTML
<!-- index.html -->
<head>
  <link rel="stylesheet" href="style.css">
</head>
```

2. **შიდა (Internal) CSS**

   გამოიყენება კონკრეტული, ერთი HTML გვერდის სტილიზაციისთვის.

> **როგორ:** გამოიყენეთ `<style>` ტეგები `<head>` სექციაში.

```HTML
<head>
  <style>
    body {
      background-color: lightgray;
    }
  </style>
</head>
```

3. **ინლაინ (Inline) CSS**

   გამოიყენება პირდაპირ ელემენტზე სტილის მისანიჭებლად.

   მოერიდეთ ამ მეთოდს, თუ აუცილებელი არ არის, რადგან მისი მართვა რთულია.

> **როგორ:** გამოიყენეთ `style` ატრიბუტი პირდაპირ HTML ტეგში.

```HTML
<h1 style="color: blue;">გამარჯობა მსოფლიო</h1>
```

## ნაწილი 2: სელექტორები და სინტაქსი

| სელექტორის ტიპი | სიმბოლო   | მაგალითი                 | აღწერა                                                                      |
| --------------- | --------- | ------------------------ | --------------------------------------------------------------------------- |
| უნივერსალური    | \*        | { margin: 0; }           | ნიშნავს ყველა ელემენტს გვერდზე.                                             |
| HTML-ის ტეგი    | (არ აქვს) | p { color: red; }        | ნიშნავს ამ ტიპის ყველა ელემენტს (მაგ: ყველა პარაგრაფს).                     |
| კლასი (Class)   | .         | .btn { color: blue; }    | ნიშნავს ელემენტებს კონკრეტული კლასის სახელით. მრავალჯერადი გამოყენებისთვის. |
| ID              | #         | #header { width: 100%; } | ნიშნავს ერთ უნიკალურ ელემენტს კონკრეტული ID-ით.                             |

## ნაწილი 3: სპეციფიკურობა (წონა)

როდესაც ერთსა და იმავე ელემენტზე ვრცელდება რამდენიმე წესი, ბრაუზერი წყვეტს რომელი გამოიყენოს **"წონის" (Weight)** მიხედვით. იმარჯვებს ის, ვისაც **მეტი წონა** აქვს.

### იერარქია ზრდადობით

1. უნივერსალური სელექტორი (\*) (ყველაზე დაბალი წონა: 0)

1. ტეგის სელექტორები (h1, div, p)

1. კლასის სელექტორები (.box, .text-center)

1. ID სელექტორები (#main-nav)

1. ინლაინ სტილი (Inline Style) (პირდაპირ HTML-ში გაწერილი)

1. !important (ყველაფერს გადაწონის)

### როგორ ითვლება წონა?

ვისაც მეტი ქულა აქვს, ის იმარჯვებს.

| სელექტორი           | კალკულაცია (ID - Class - Tag)  | სრული წონა |
| ------------------- | ------------------------------ | ---------- |
| h1                  | 0 - 0 - 1                      | 1          |
| h1.title            | 0 - 1 - 1                      | 11         |
| .card .title        | 0 - 2 - 0                      | 20         |
| #navbar             | 1 - 0 - 0                      | 100        |
| `<div style="...">` | (Inline ყოველთვის ჯობნის ID-ს) | 1000       |

## შენიშვნა !important-ზე

> თუ მნიშვნელობას დაუმატებთ !important-ს, ის დაარღვევს სპეციფიკურობის წესებს და იძულებით მიანიჭებს სტილს. გამოიყენეთ იშვიათად, რადგან ეს ართულებს კოდის მართვას და დებაგინგს.

```TS
p {
    color: red !important; /* იქნება წითელი, მაშინაც კი, თუ ID სხვა ფერს ითხოვს */
}
```

## შემაჯამებელი მაგალითი

როგორ წყვეტს ბრაუზერი რა ფერი ექნება პარაგრაფს:

```HTML
<p id="unique" class="text" style="color: purple;">გამარჯობა სტუდენტებო!</p>
```

```CSS
/_ წონა: 0-0-1 (ძალიან სუსტი) _/
p { color: black; }

/_ წონა: 0-1-0 (ტეგზე ძლიერია) _/
.text { color: green; }

/_ წონა: 1-0-0 (კლასზე ძლიერია) _/
#unique { color: blue; }
```

> შედეგი: ტექსტი იქნება იასამნისფერი (Purple).
> რატომ? მიუხედავად იმისა, რომ #unique მძიმეა, HTML-ში გაწერილ ინლაინ სტილს (style="color: purple;") უფრო მეტი პრიორიტეტი აქვს, ვიდრე ნებისმიერ გარე CSS სელექტორს.
